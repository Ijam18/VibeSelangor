# 03 - Stage A Syllabus: 1-Week Builder Sprint

## Delivery Blueprint
- Day 1: 2-hour publish-with-me live workshop
- Days 2-6: troubleshooting + feature completion clinics
- Day 7: review and demo day
- Attendance: Day 1 and Day 7 only

## Day-by-Day Plan
## Day 1 (120 min)
Goals:
- Scaffold app
- Push project
- Deploy first live version

Agenda:
1. 0-10 min: kickoff, outcomes, setup checks
2. 10-25 min: app scaffold
3. 25-45 min: push to version control
4. 45-70 min: deployment walkthrough
5. 70-90 min: learner follow-along publish
6. 90-110 min: common blockers and fixes
7. 110-120 min: exit ticket

Evidence:
- live URL
- project link (if available)

## Day 2
Focus:
- M1 problem framing refinement
- M2 flow cleanup
Deliverable:
- one-page problem brief + updated user flow

## Day 3
Focus:
- M3 core features
Deliverable:
- 1 complete end-to-end user flow

## Day 4
Focus:
- M3 data/connectivity integration
Deliverable:
- create/read flow works in demo

## Day 5
Focus:
- M4 test and verify
Deliverable:
- QA checklist completed

## Day 6
Focus:
- M5 iterate from feedback
Deliverable:
- release notes (v0.2)

## Day 7
Focus:
- M6 3-minute demo and rubric evaluation
Deliverable:
- demo recording or live presentation
- rubric score and progression decision

## Daily Non-Attendance Artifact (Days 2-6)
Each learner submits:
1. What changed today
2. Main blocker
3. Current live URL
4. Next action
