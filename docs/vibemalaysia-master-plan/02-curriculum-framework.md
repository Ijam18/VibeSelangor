# 02 - Curriculum Framework (All Ages)

## Curriculum Spine (M0-M6)
### M0 Builder Mindset & Safety
Outcome:
- Identity shift from user to builder
- AI ethics, cyber hygiene, safe prompting
Artifacts:
- AI usage agreement
- Safe prompt checklist

### M1 Problem Framing
Outcome:
- Define user, pain point, and value proposition
Artifacts:
- One-page problem brief

### M2 Prompt-to-Product Design
Outcome:
- Build user flow and screen architecture
Artifacts:
- 3-screen blueprint
- Design board

### M3 Build & Integrate
Outcome:
- Working app with core logic and data path
Artifacts:
- Functional prototype

### M4 Deploy & Verify
Outcome:
- Public deployment and quality verification
Artifacts:
- Live URL
- QA checklist results

### M5 Validate & Improve
Outcome:
- Learn from users and improve app
Artifacts:
- Feedback log
- Release improvements

### M6 Pitch & Pathway
Outcome:
- Demonstrate product value and next steps
Artifacts:
- 3-minute demo
- Growth roadmap

## Track-Specific Pedagogy
## Juniors (9-12)
- Delivery: guided templates, visual blocks, short bursts
- Session length: max 90 minutes/day
- Sprint output: interactive mini app/story/game
- Cohort output: themed app with one clear user scenario

## Youth (13-24)
- Delivery: studio mode, peer critique, challenge loops
- Session length: 2-3 hrs/day in sprint; 2 sessions/week in cohort
- Sprint output: deployable web app
- Cohort output: MVP with analytics + user interviews

## Adults/SMEs (25+)
- Delivery: ROI-first, practical automation use-cases
- Session length: evening/weekend-friendly blocks
- Sprint output: deployable business utility app
- Cohort output: lead-gen/ops MVP with measurable KPI

## Learning Outcome Equivalency (Cross-Track)
All tracks must prove:
1. Functionality: app works for intended user action
2. Design quality: clear, usable, mobile-friendly experience
3. Potential: problem-solution fit and scale path
