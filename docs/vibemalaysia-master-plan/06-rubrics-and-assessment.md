# 06 - Rubrics and Assessment (No-Codes-Only Vibes)

## Standard Rubric (100 Points)
1. Functionality (45)
- End-to-end core flow works
- Stable behavior in live environment
- No critical blocking errors

2. Design & UX (30)
- Clarity of navigation and content
- Visual consistency and accessibility basics
- Mobile usability

3. Potential (25)
- Real problem fit
- User adoption likelihood
- Growth/scalability path

## Pass Thresholds
- Stage A completion: >= 60
- Stage B certification: >= 70
- Stage C eligibility: >= 75 plus traction evidence

## Track Weight Overrides
## Juniors
- Functionality 40
- Design clarity 35
- Potential/creativity 25

## Youth
- Functionality 45
- Design & UX 30
- Potential/problem fit 25

## Adults/SMEs
- Functionality 45
- Design trust/usability 25
- Potential/business impact 30

## Attendance Policy
- Formal attendance is recorded only on Day 1 and Day 7 (Sprint)
- Days 2-6 are tracked via progress artifacts

## Rating Anchors
- Excellent: clear, reliable, and demo-ready
- Good: mostly solid with minor gaps
- Needs Work: incomplete core flow or unclear value

## QA Scenarios
- A: live app exists but core flow broken -> fail functionality gate
- B: app works, UX weak -> conditional pass with improvement plan
- C: strong app + validated users -> fast-track recommendation
- D: junior creativity high, complexity low -> evaluate by junior rubric
- E: SME KPI impact clear -> potential/business score uplift
